Project: Anonymous Suggestion / Voting App

We are building an application where members of an organization can submit suggestions or feedback anonymously. Only users with an organization email (for example @sece.ac.in) can access the application. The identity of the submitter is stored for moderation/audit purposes only and is not shown inside the app. This allows honest feedback to surface while still allowing the organization to block misusers.

  • Post anonymous suggestions (identity stored in a secure collection, not displayed)
  • Vote on suggestions so important issues bubble up
  • Abusive users can be identified by admins and blocked — the app doesn't show identity publicly

How it works (short)

  1. Sign in with your organization email (must end with @sece.ac.in).
  2. After sign-in you can open the suggestion board and post anonymously.
  3. Admins can find the user behind an abusive post via secure logs and restrict their access.

Next / Admin

If you're an admin and need to manage users, edit the "allowedUsers" collection in Firestore (document IDs are emails).
